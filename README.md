## 🔍 Что такое Docker?

Docker позволяет упаковать приложение со всеми его зависимостями (библиотеки, конфигурационные файлы, переменные окружения) в единый объект — **контейнер**.

### Ключевые отличия от виртуальных машин (VM):
| Характеристика | Виртуальная машина (VM) | Контейнер (Docker) |
| :--- | :--- | :--- |
| **Изоляция** | Полная (включая ядро ОС) | Процессная (общее ядро ОС хоста) |
| **Вес** | Тяжелые (ГБ) | Легкие (МБ) |
| **Запуск** | Минуты | Секунды |
| **Производительность** | Ниже (оверхед гипервизора) | Ближе к нативной |

**Главная цель Docker:** решить проблему *"У меня на машине работает, а на сервере нет"*.

## 📜 История появления и развития

### Предыстория (до 2013)
Технологии изоляции процессов существовали давно:
*   **1979:** `chroot` в Unix.
*   **2000-е:** FreeBSD Jails, Linux VServer, LXC (Linux Containers).
*   Однако эти инструменты были сложны в настройке и использовании для массового разработчика.

### Рождение Docker (2013)
*   **Март 2013:** Компания **dotCloud** (Париж) представляет Docker как open-source проект на конференции PyCon.
*   **Идея:** Сделать контейнеры удобными для разработчиков, абстрагировав сложность LXC.
*   **Ключевое лицо:** Соломон Хайкс (Solomon Hykes), основатель dotCloud.

### Взлет и стандартизация (2014–2016)
*   **2014:** Создана компания **Docker, Inc.** Проект отделяется от dotCloud.
*   Внедрение формата образов и реестра (Registry).
*   Docker становится стандартом де-факто для упаковки микросервисов.
*   Начало "войн оркестраторов": Docker Swarm vs Kubernetes vs Mesos.

### Зрелость и разделение (2017–2019)
*   **2017:** Docker передает спецификацию контейнеров и рантайм (containerd) в CNCF (Cloud Native Computing Foundation).
*   **2019:** Бизнес-подразделение **Docker Enterprise** продано компании **Mirantis**. Docker, Inc. сосредоточилась на инструментах для разработчиков (Docker Desktop).

## 🚀 Современное состояние

На сегодняшний день Docker является фундаментальной технологией в мире Cloud Native.

*   **Индустриальный стандарт:** Почти все CI/CD пайплайны (GitLab CI, GitHub Actions, Jenkins) используют Docker для сборки и тестирования.
*   **Оркестрация:** В продакшене одиночные контейнеры используются редко. Обычно они управляются оркестраторами, где лидером является **Kubernetes (K8s)**. Docker Engine часто используется для локальной разработки, а в кластерах K8s используется containerd или CRI-O.
*   **Безопасность:** Усилен фокус на сканирование образов на уязвимости (Docker Scout, Trivy).
*   **Конкуренция:** Появились альтернативы, такие как **Podman** (daemonless, rootless), но Docker сохраняет лидерство по удобству DX (Developer Experience).

## 📦 Docker Hub и готовые образы

**Docker Hub** — это облачный реестр для хранения и распространения Docker-образов. Аналог GitHub для контейнеров.

### Типы образов:
1.  **Official Images:** Поддерживаются производителями ПО или сообществом Docker (например, `nginx`, `python`, `postgres`). Наиболее безопасны.
2.  **Verified Publisher:** Образы от проверенных компаний.
3.  **User Images:** Созданы любыми пользователями (требуют проверки перед использованием в продакшене).

### Основные команды:
```bash
# Поиск образа
docker search nginx

# Скачивание образа
docker pull nginx:latest

# Запуск контейнера из образа
docker run -d -p 80:80 nginx

# Отправка своего образа в реестр
docker push myusername/myapp:1.0
```

## 📝 Dockerfile

**Dockerfile** — это текстовый файл с инструкциями по сборке образа. Каждая инструкция создает новый слой образа, что позволяет эффективно использовать кэш.

### Пример Dockerfile (Python приложение):

```dockerfile
# Базовый образ
FROM python:3.9-slim

# Установка рабочей директории
WORKDIR /app

# Копирование зависимостей и установка
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Копирование исходного кода
COPY . .

# Переменная окружения
ENV FLASK_APP=main.py

# Порт, который слушает приложение
EXPOSE 5000

# Команда запуска
CMD ["python", "-m", "flask", "run", "--host=0.0.0.0"]
```

### Основные инструкции:
*   `FROM`: Базовый образ.
*   `RUN`: Выполнение команды в shell (установка пакетов).
*   `COPY` / `ADD`: Копирование файлов в образ.
*   `ENV`: Установка переменных окружения.
*   `CMD`: Команда, выполняемая при запуске контейнера (можно переопределить).
*   `ENTRYPOINT`: Основная исполняемая программа (сложнее переопределить).

## 🎼 Docker Compose

**Docker Compose** — это инструмент для определения и запуска многоконтейнерных приложений Docker. Использует файл `docker-compose.yml`.

Идеально подходит для локальной разработки, когда приложению нужны база данных, кэш и бэкенд одновременно.

### Пример docker-compose.yml:

```yaml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "5000:5000"
    depends_on:
      - db
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/mydb

  db:
    image: postgres:13
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: mydb
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:
```

### Основные команды:
```bash
# Запуск всех сервисов
docker-compose up -d

# Просмотр логов
docker-compose logs -f

# Остановка и удаление контейнеров
docker-compose down
```
